ChatGPT 的三種模式
Chat · Work · Codex

ChatGPT 早就不只是「聊天」了。它現在分成三種模式,對應三種需求:產出內容、完成工作、開發程式。搞懂差別,才知道什麼時候該開哪一個。

一句話看懂三種模式

MODE 01

Chat

對話 · 把想法變成內容
  • 問答、寫文案、翻譯
  • 摘要、腦力激盪
  • 學習、查詢、發想
👤 適合:每個人的日常
📤 產出:文字對話
MODE 02

Work

工作 · 把一整套工作做完
  • 串接工具、讀你的檔案
  • 跑完整工作流、產出成品
  • 一句話生出網站(Sites)
👤 適合:要交出成果的上班族/團隊
📤 產出:完成的任務/檔案/網站
MODE 03

Codex

開發 · 管理程式碼、寫程式
  • 讀懂整個專案(codebase)
  • 幫你寫、改、測程式
  • 自動化開發任務
👤 適合:工程師 / 開發團隊
📤 產出:程式碼、修改建議
核心差別:Chat 幫你「想與寫」、Work 幫你「做完一件事」、Codex 幫你「動程式碼」。能力一個比一個「能動手」。

你該用哪一個?(決策指南)

我只是要產出內容 —— 寫文案、摘要、發想、翻譯
→用 Chat
我要完成一項工作、交出成品 —— 整合資料、跑流程、生網站或報告
→用 Work
我要開發軟體、動到程式碼 —— 讀專案、改程式、跑開發任務
→用 Codex

詳細比較表

模式一句話定位適合誰適合任務產出形式
Chat把想法變成內容所有人問答、寫作、翻譯、發想文字對話
Work把一整套工作做完要交付成果的上班族、團隊工作流、整合工具、生成成品/網站任務、檔案、網站
Codex管理與撰寫程式碼工程師、開發團隊讀改程式、開發任務、專案管理程式碼、PR/修改

亮點功能:一句話生網站(ChatGPT Sites)

在 Work 模式裡,有一個很有感的功能:用一句話描述,就能生出一個網站。

例:「幫我做一個咖啡店的一頁式介紹網站,有菜單和聯絡方式」→ 幾秒後就有可看的成品。這正是我們課程「用 AI 做網站」的同一個概念 —— 描述 → 生成 → 上線。

模型 vs 產品(最多人搞混的地方)

模型是「引擎」(例如 GPT-5.6 世代);Chat / Work / Codex 是「產品/使用模式」(你怎麼開這台引擎)。

同一代模型,可以跑在三種模式上;差別在「它被允許做到什麼程度」—— Chat 只回話、Work 能動工具、Codex 能碰程式。

提醒:各家的模型版本、方案價格、可用模式變動很快 —— 實際能用哪些、要不要付費,請以 OpenAI 官方當天頁面為準。

使用限制與權限(用之前先知道)

🎟️ 用量額度

Work / Codex 這類「會動手」的模式,通常有使用次數/額度上限,免費與付費差很多。

🔑 授權

要它讀你的檔案、串接工具,需要你授權;授權範圍越大,能做的越多,也越要小心。

🛡️ 資安

公司敏感資料要不要餵給它,先確認規範。能動手的 AI,權限就是風險點。

常見問題 FAQ

免費版能用嗎?
Chat 免費就能用;Work / Codex 多為付費或有額度限制。以官方當天為準。
三種到底差在哪?
一句話:Chat 產內容、Work 把工作做完、Codex 寫程式。能「動手」的程度依序增加。
這跟 Claude 的「Cowork」有什麼不同?
概念類似 —— 都是讓 AI 從「會回答」進化到「能做事」。各家做法與生態不同,重點是選你團隊已經在用的那一套,別為了工具而工具。
GPT-5.6 是模型還是產品?
是「模型」(引擎)。ChatGPT / Work / Codex 是「產品/模式」(你怎麼用它)。別把兩者混在一起。

一句話小結

選模式不用背規格,記住一句就好:
想與寫 → Chat　·　做完一件事 → Work　·　動程式碼 → Codex。
